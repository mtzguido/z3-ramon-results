Z3 commit message: seq_regex_bisim: classical-leaf inequivalence shortcut

Adds an early-exit decision to are_equivalent_core based on the
get_info().classical flag.  A classical regex (built only from
str.to_re, re.all, concat, union, star, plus, opt, loop, orelse)
always has non-empty language, so:

  - A bare leaf l of the bisim worklist represents the implicit
    pair (empty XOR l); if l is classical, L(l) is non-empty and
    yields a distinguishing prefix => return l_false.

  - For an XOR leaf xor(a, b), if both sides are classical and
    have different min_length, the shortest word of one is not
    in the other => return l_false.

The check is applied to r0 immediately after mk_re_xor_simplified
(in case the AC reduction collapsed p XOR q to a bare classical
residual) and to every leaf produced by collect_leaves.

A new statistic 'seq bisim classical-cut' counts the number of
queries decided this way.
