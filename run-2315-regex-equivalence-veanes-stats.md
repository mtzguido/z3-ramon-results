Z3 commit message: seq_regex_bisim: use Brzozowski derivative and fix mk_re_xor0 distribution

Two related fixes for soundness and performance of the XOR-based regex
bisimulation introduced in PR #9804.

1. **Brzozowski derivative for the bisim.** The bisimulation procedure
   was calling `seq_rewriter::mk_derivative` (the public entry point),
   which invokes `mk_antimirov_deriv` and lifts unions to
   `_OP_RE_ANTIMIROV_UNION` at the top of the derivative tree.
   `collect_leaves` descends through `is_antimirov_union` like any
   other union, which breaks the invariant that every leaf of
   `D(p XOR q)` is a coherent XOR pair `(D_v p) XOR (D_v q)`. Without
   that invariant `merge_leaf` produces unsound merges and the
   procedure reports `l_true` for inequivalent regexes (observed e.g.
   on `(.*ba.*){n}` vs `(.*ab.*){n}` with n >= 2).

   `mk_derivative_rec` is reworked to be a true Brzozowski derivative:
   - All internal recursive calls now go through `mk_derivative_rec`
     itself instead of `mk_derivative` (so antimirov lifting is never
     re-introduced from inside).
   - At the concat case the antimirov union is replaced with the plain
     `mk_der_union`, keeping the result as a single ITE/BDD tree.

   A new public entry point `seq_rewriter::mk_brz_derivative(expr*)`
   wraps the recursive function, mirroring `mk_derivative(expr*)` but
   producing the classical (single-tree) derivative. The bisim now calls
   this entry point. `mk_derivative` and `mk_antimirov_deriv` are
   unchanged and still used by the rest of theory_seq (e.g. for
   `str.in_re` axiomatisation).

2. **mk_re_xor0: reduction-only XOR pre-simplification for mk_der_op.**
   `mk_der_op` pre-simplifies operands with the corresponding
   `mk_re_*` rewrite before distributing through ITE BDDs. The full
   `mk_re_xor` does AC normalisation (id-based operand swap returning
   BR_DONE with a fresh `re.xor` term), which prevents the subsequent
   BDD distribution in `mk_der_op_rec`. The result was a top-level
   `(re.xor (ite ...) (ite ...))` leaf that the bisim could not
   evaluate (`is_nullable` returned a conditional boolean over the
   symbolic character variable, defeating `nullability` and bailing
   `l_undef`).

   A new `mk_re_xor0` (mirroring `mk_re_union0` and `mk_re_inter0`)
   is used inside `mk_der_op` for OP_RE_XOR. It applies only the safe
   reductions (`a == b`, `is_empty(a)`, `is_empty(b)`) and never
   creates an `re.xor` node. AC normalisation and complement absorption
   still happen in the leaf case at the bottom of `mk_der_op_rec`,
   where both operands are already non-ITE regex leaves.

**Stats infrastructure.** `regex_bisim` now exposes a thread-local
`stats_t` and a `collect_statistics` static so `-st` reports
`seq bisim queries` / `decided` / `undef` / `undef-*` /
`steps` / `time`. `theory_seq::collect_statistics` forwards the
call.
